Encryption

Data with no encryption is plaintext data. This data can be read by anyone who sees it. Encryption is the “scrambling” or ciphering of data before transmission. In short, the data in a message goes through a calculated process (or algorithm) of turning the information to be sent into an unreadable format. The purpose of this is to protect the information that is being sent from being deciphered by an unauthorized third-party or a bad actor who may capture that information. Decryption is the process of deciphering an encrypted message using a key or set of steps in order to turn the unreadable content into comprehendible information (plaintext to the recipient). In a perfect scenario, the intended recipient is the only holder of the key.

Symmetric Encryption

Symmetric encryption implies that the encryption and decryption method is using the same key or steps to secure messages. The sender and the receiver would be using the same key.

Asymmetric Encryption

Asymmetric encryption implies that the sender uses a public key to encrypt a transmission. The recipient uses a private key (different than the sender’s key) to decrypt the transmission that was sent. The sender’s key being different, prevents them from decrypting the key that they sent. With the recipient's device being the only one that can decrypt the transmission, this makes it very difficult for anyone else to decipher the encryption. HTTPS utilizes this method.

The Importance of Encryption

Compared to the internet that was derived from the ARPANET decades ago, the internet is now used to fulfill services that use personally identifiable information or financial information of millions, if not, billions of people around the world. With all of the exchange of this information, bad actors, hackers, and so on, want to access this information for illicit reasons. In order to prevent this, security measures need to be taken in how this traffic is sent across the web. Encrypting the information being sent to other devices is a measure that helps ensure that not anyone can intercept confidential information and make use of it in any meaningful way. This would be similar to intercepting an envelope from the mail to an important person but not being able to make sense of the message because it’s literally unreadable.

HTTPS

In the early days of the internet, HTTP was used to request and load web pages on a user's internet browser. At the time of conception, data security wasn't yet an issue when on the web. When issues about the security of how data was sent arose, HTTPS was developed to mitigate possible issues with intercepted data. HTTPS utilizes SSL/TLS certificates for websites to use with their users requesting an exchange of data.

SSL/TLS Encryption

Secure Sockets Layer (SSL) or Transport Layer Security uses a public Key to encrypt the traffic between the website origin and the user. The purpose of this is to mitigate a Man-In-The-Middle (MITM) attack, which involves someone having access to a middle device in the exchange of information and capture the data. This poses an issue for money transactions and exchange of confidential information.
